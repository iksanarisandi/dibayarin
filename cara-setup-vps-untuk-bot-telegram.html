Cara Setup VPS untuk Bot Telegram: Panduan Lengkap dari Install hingga 24/7 Running

Panduan lengkap cara setup VPS untuk bot Telegram dari awal sampai bot berjalan 24/7. Mulai dari pemilihan VPS, install dependensi, deploy bot Python/Node.js, sampai konfigurasi auto-restart dengan PM2 atau systemd.

Prerequisites

  • Bot Telegram: Sudah dibuat via @BotFather
  • Bot Token: Sudah dapat dari @BotFather
  • VPS: Ubuntu 20.04/22.04 atau Debian (minimal 512MB RAM, 1GB recommended)
  • Basic Linux: Familiar dengan SSH dan command line

Langkah 1: Setup VPS

1.1 Sewa VPS

Pilih provider VPS yang sesuai budget dan lokasi. Rekomendasi untuk bot Telegram:

  • AWS Lightsail: $3.50/bulan (512MB RAM) - Paling murah
  • Linode: $5/bulan (1GB RAM, Singapore) - Balance choice
  • DigitalOcean: $6/bulan (1GB RAM) - User-friendly

1.2 Connect ke VPS via SSH

  • Dapatkan IP address VPS dari provider
  • Connect: ssh root@your-vps-ip
  • Masukkan password atau gunakan SSH key

1.3 Initial Setup

  • Update system: apt update && apt upgrade -y
  • Setup firewall: ufw allow OpenSSH && ufw enable
  • Create non-root user (recommended): adduser botuser && usermod -aG sudo botuser

Langkah 2: Install Dependensi

2.1 Install Python (untuk Bot Python)

  • Python 3 biasanya sudah terinstall: python3 --version
  • Jika belum: apt install python3 python3-pip -y
  • Install virtual environment: apt install python3-venv -y

2.2 Install Node.js (untuk Bot Node.js)

  • curl -fsSL https://deb.nodesource.com/setup_18.x | bash -
  • apt install nodejs -y
  • Verify: node --version && npm --version

2.3 Install Git

  • apt install git -y

Langkah 3: Deploy Bot Telegram

3.1 Clone Bot Code

  • Switch ke bot user: su - botuser
  • Clone repository: git clone https://github.com/your-username/your-telegram-bot.git
  • Enter directory: cd your-telegram-bot

3.2 Setup Environment Variables

  • Buat file .env: nano .env
  • Add konfigurasi:
    • TELEGRAM_BOT_TOKEN=your_bot_token_from_botfather
    • API_KEY=your_api_key_if_needed
    • DATABASE_URL=your_database_url_if_needed
  • Save dan exit (Ctrl+X, Y, Enter)

3.3 Install Dependencies

Untuk Python Bot:

  • Create venv: python3 -m venv venv
  • Activate: source venv/bin/activate
  • Install: pip install -r requirements.txt

Untuk Node.js Bot:

  • npm install

3.4 Test Run Bot

Python: python3 bot.py
Node.js: npm start

Buka Telegram, kirim pesan ke bot. Jika bot merespon, berarti berhasil!

Tekan Ctrl+C untuk stop bot (kita akan setup auto-start nanti).

Langkah 4: Setup Process Manager (PM2)

PM2 memastikan bot auto-restart jika crash dan auto-start setelah VPS reboot.

4.1 Install PM2

  • npm install -g pm2

4.2 Start Bot dengan PM2

Untuk Python Bot:

  • pm2 start python3 --name telegram-bot -- bot.py

Untuk Node.js Bot:

  • pm2 start npm --name telegram-bot -- start
  • Atau jika ada start script: pm2 start bot.js --name telegram-bot

4.3 Configure PM2 Startup

  • Save PM2 config: pm2 save
  • Generate startup script: pm2 startup
  • Copy dan jalankan command yang diberikan PM2

4.4 Monitor Bot dengan PM2

  • Cek status: pm2 status
  • Lihat logs: pm2 logs telegram-bot
  • Restart bot: pm2 restart telegram-bot
  • Stop bot: pm2 stop telegram-bot

Langkah 5: Alternative - Systemd Service

Alternatif PM2, gunakan systemd (native Linux):

5.1 Create Service File

  • sudo nano /etc/systemd/system/telegram-bot.service
  • Add konfigurasi (sesuaikan dengan path bot Anda): 
    [Unit]
Description=Telegram Bot
After=network.target

[Service]
Type=simple
User=botuser
WorkingDirectory=/home/botuser/your-telegram-bot
ExecStart=/home/botuser/your-telegram-bot/venv/bin/python3 bot.py
Restart=always

[Install]
WantedBy=multi-user.target

5.2 Enable dan Start Service

  • Reload systemd: sudo systemctl daemon-reload
  • Enable service: sudo systemctl enable telegram-bot
  • Start service: sudo systemctl start telegram-bot
  • Cek status: sudo systemctl status telegram-bot

Troubleshooting

Bot tidak merespon

Check logs: pm2 logs telegram-bot atau journalctl -u telegram-bot. Verify bot token correct, cek internet connection VPS.

Bot crash terus

Check error logs, fix bugs di code. PM2 akan auto-restart, tapi fix root cause agar tidak loop.

VPS tidak bisa clone dari GitHub

Setup SSH keys atau gunakan personal access token. Atau upload code manual via SCP.

Scaling untuk Multiple Bots

Satu VPS bisa host multiple bot Telegram:

  • Deploy multiple bot di satu VPS dengan PM2
  • Atau gunakan Docker untuk containerization
  • Each bot run sebagai separate PM2 process
  • Monitor resource usage, upgrade VPS jika needed
